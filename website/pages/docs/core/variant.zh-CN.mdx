# 修改和扩展颜色种类

## 修改原有颜色

```js filename="icestack.config.js"
/**
 * @type {import('@icestack/ui').Config}
 */
const config = {
  outdir: './my-ui',
  // 修改 base
  base: {
    // 主题
    themes: {
      // 覆盖 light mode 下的颜色
      light: {
        types: {
          primary: '#eb2f96'
        }
      },
      // 同样也可以覆盖 dark mode 下的表现
      dark: {
        types: {
          primary: '#cb2b83'
        }
      }
    }
  }
}
module.exports = config
```

你传入一个颜色字符串，它会被交给 [`ant-design-colors`](https://github.com/ant-design/ant-design-colors) 进行颜色的自动生成，从而计算出 `hover`,`active` 等状态的颜色

当然你也可以传入一个 `object` 来精准的控制所有状态下的颜色：

```js filename="icestack.config.js"
/**
 * @type {import('@icestack/ui').Config}
 */
const config = {
  outdir: './my-ui',
  // 修改 base
  base: {
    // 主题
    themes: {
      // 覆盖 light mode 下的颜色
      light: {
        types: {
          primary: {
            // 一共四种颜色，key 的名称就是 css 变量名
            primary: '#1677ff',
            'primary-content': '#ffffff',
            // 主要是上面2种颜色， hover 和 active 色是给部分组件使用的，比如 `button`
            'primary-hover': '#4096ff',
            'primary-active': '#0958d9'
          }
        }
      }
      // dark:{}
    }
  }
}
module.exports = config
```

修改完成之后执行 `icestack build` 进行构建即可。

## 扩展更多颜色

```js filename="icestack.config.js"
/**
 * @type {import('@icestack/ui').Config}
 */
const config = {
  outdir: './my-ui',
  // 修改 base
  base: {
    themes: {
      light: {
        // 或者你添加一种新的颜色主题: pure
        types: {
          // light mode colors
          pure: '#1677ff'
        }
      },
      dark: {
        types: {
          // dark mode colors
          pure: '#642ab5'
        }
      }
    }
  }
}
module.exports = config
```

## 构建

再修改好配置之后，执行 `icestack build` 指令, 重新构建 `my-ui`

接着你会发现，新的颜色主题，和原先主题色的覆盖，相关的改动就已经出现新的 `my-ui` 产物中了，

这时候你就可以使用:

```js
'btn-pure', 'badge-pure'
```

这种类名了。
