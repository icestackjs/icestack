# 首页

## 前言

如你从这个库名可以提取到的信息一样，这又是一个该死的 UI 框架。

现如今 UI 框架们已经繁如星海，有些已经在业界占据了稳固的位置了，就像 `antd`/`element`，几乎变成了 `react`/`vue`中首先考虑的框架。

`@icestack/ui` 究竟有什么特别之处呢？

## 什么是 `@icestack/ui`

`@icestack/ui` 本质上不是一个 UI 框架，它的本质是一个 `UI CSS 范式` 生成器。

也就是通过它，你可以快速的生成一套属于你自己的 `CSS 范式`, 当然，你也可以默认使用我为你提供的一套默认 `范式`。

> 假如打个比方，默认 `范式` 就像 `axios` 默认实例, 你自己生成的范式那就是 `axios.create`

你可以根据配置项，自由的去操纵所有 `CSS`范式的表现性状，包括不但限于:

- 添加/减少/覆盖/修改主题色给不同的组件 (比如: 按钮默认有 `default`, `primary`,`success`,`warning`,`error`, 你再去添加个`secondary`,`accent`色，并声明它们`hover/focus/active`等等状态)
- 添加/减少/覆盖/修改组件的大小，形状，样式 (比如:按钮默认 `sm`,`md`,`lg` 你再添加 `xs`,`xl`,`2xl`的大小)
- 添加新的`CSS`组件到这套体系中来

而且，这个生成器生成的 `CSS 范式`结果入侵性非常小，默认情况下，它不会携带任何的 `css reset/normalize` 样式，只会默认注入一些 `css` 变量。

它的生成结果也非常的直观，你可以自定义出想要的 `UI 框架` 后进行二次封装，或者提取成 `preset`, 从而在多个项目中共享。

> 这个框架是 `重编译时` + `重 tailwindcss 插件运行时`，最终到你的系统里就变成了 `0 运行时`。

## 使用方式

`@icestack/ui` 本身可以作为 `generator cli` 来使用，也可以作为 `tailwindcss plugin` 来使用，更多的使用方式正在开发中
