## 什么是 `@icestack/ui`

`@icestack/ui` 本质上不是一个 UI 框架，它的本质是一个 `UI CSS 范式` 管理生成器。

比如 `tailwindcss` 里最流行的框架是 `daisyui`，你可以通过 `@icestack/ui` 定制生成无数个属于你自己的 `daisyui`

例如我们可以利用它，构建出一套 `daisyui` 的适配微信小程序的版本([点击此处跳转演示小程序](./usage/mp#演示小程序)):

{/* ![icestack](/mp.jpg) */}

没错！也就是通过它，你可以快速的生成一套属于你自己的 `CSS` 框架！当然你可以使用内部提供的一套默认 `范式`,也可以对它进行覆盖或者扩展。

你可以根据配置项，自由的去操纵所有 `CSS`范式的表现性状，包括不但限于:

- 添加/减少/覆盖/修改 主题色给不同的组件 (比如: 按钮默认有 `default`, `primary`,`success`,`warning`,`error`, 你再去添加个`secondary`,`accent`色，并声明它们`hover/focus/active`等等状态)
- 添加/减少/覆盖/修改组件的大小，形状，样式 (比如:按钮默认的尺寸有`xs`,`sm`,`md`,`lg` 四种尺寸，你可以再往里面添加 `xl`,`2xl` 等等的大小尺寸)
- 添加新的`CSS`组件到这套体系中来。

而且，这个生成器生成的 `CSS 范式` 结果入侵性非常小，默认情况下，它不会携带任何的 `css reset/normalize/preflight` 样式，只会全局注入一些计算出的 `CSS` 变量。

它的生成结果也非常的直观，你可以使用它自定义出想要的 `UI` 行为后，进行二次封装，或者提取成 `preset`, 从而在多个项目中共享。
