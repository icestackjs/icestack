# 修改和扩展颜色种类

作为一名开发者，在设计我们自己应用的时候，很少有对第三方控件库默认的组件样式感到满意的

在 `@icestack/ui` 的默认预设下:

## 颜色

颜色类型主要有五种分别为 `primary`,`success`,`warning`,`error`,`neutral`

于是 `css` 产物中就生成出了

```js
// btn
'btn-primary', 'btn-success', 'btn-warning', 'btn-error', 'btn-neutral'

// badge
'badge-primary', 'badge-success', 'badge-warning', 'badge-error', 'badge-neutral'
```

这类的css产物

## 尺寸形状

同样的原理，默认尺寸有 `xs`,`sm`,`md`,`lg` 所以会生成出:

```js
// btn
'btn-xs', 'btn-sm', 'btn-md', 'btn-lg',

// badge
'badge-xs', 'badge-sm', 'badge-md', 'badge-lg'
```

## 主题

默认有 `light` 和 `dark`，它们会生成出各自的 `CSS` 变量块，当然你也可以对它们进行覆盖和扩展。

## 修改和扩展配置

```js filename="icestack.config.js"
/**
 * @type {import('@icestack/ui').Config}
 */
const config = {
  outdir: './my-ui',
  // 修改 base
  base: {
    // 颜色 type
    types: {
      // override primary type
      // 覆盖 primary 色
      primary: {
        // 覆盖 light mode 下的颜色
        light: {
          // 一共四种颜色，key 的名称就是 css 变量名
          primary: '#1677ff',
          'primary-content': '#ffffff',
          // 主要是上面2种颜色， hover 和 active 色是给部分组件使用的，比如 `button`
          'primary-hover': '#4096ff',
          'primary-active': '#0958d9'
        }
        // 同样也可以覆盖 dark mode 下的表现
        // dark: {}
      },
      // 或者你添加一种新的颜色主题: pure
      pure: {
        // light mode colors
        light: {
          pure: '#1677ff',
          'pure-hover': '#4096ff',
          'pure-active': '#0958d9',
          'pure-content': '#ffffff'
        },
        // dark mode colors
        dark: {
          pure: '#1677ff',
          'pure-hover': '#4096ff',
          'pure-active': '#0958d9',
          'pure-content': '#ffffff'
        }
        // custom mode colors
      }
    }
  }
}
module.exports = config
```

## 构建

再修改好配置之后，执行 `icestack build` 指令, 重新构建 `my-ui`

接着你会发现，新的颜色主题，和原先主题色的覆盖，相关的改动就已经出现新的 `my-ui` 产物中了，

这时候你就可以使用:

```js
'btn-pure', 'badge-pure'
```

这种类名了。
