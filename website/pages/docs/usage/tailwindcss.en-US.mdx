# Use as tailwindcss plugin

After executing `icestack build` with the `@icestack/ui` CLI

By default `@icestack/ui` will generate all resource files locally, you need to specify a directory to store them.

For example, if we want to place the generated files in the `my-ui` folder in the current directory, we can pass an `loadDirectory` parameter like this

```sh
# yarn / pnpm
npm i -D @icestack/tailwindcss
```

```js filename="tailwind.config.js"
const path = require('node:path')
const { icestackPlugin } = require('@icestack/tailwindcss')

/** @type {import('tailwindcss').Config} */
module.exports = {
  // ...
  plugins: [
    icestackPlugin({
      loadDirectory: path.resolve(__dirname, 'my-ui'),
      // other options
      // if load theme, default: false,  when true ,config will be merge into tailwindcss.theme.extend, you can use bg-primary text-primary ....
      loadConfig: true
    })
  ]
}
```

> It is recommended to put `my-ui` into your `.gitignore` to avoid committing too many files to `git`.

Then, if we run the project normally, the code for `css` and `js` will be generated in `my-ui` in the current directory.
